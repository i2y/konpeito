# JVM VerifyError: Bad local variable type
local variable is not affected by another method defining same name
first method returns its own local value
second method returns its own local value
method parameter receives the passed argument
caller variable named same as parameter is not affected
variable assigned in if body is accessible after the if
variable assigned in else branch is accessible after the if
variable assigned in while body is accessible after the loop
nested method calls use their own independent local variables
block can read and modify variables from the enclosing scope
block parameter does not overwrite outer variable of same name after block
assignment in conditional creates variable in enclosing scope
